# lisp65 1.1.0 Release Notes

Release date: 2026-07-19

lisp65 1.1.0 is the first feature release after 1.0.1. It keeps Dialect V2 and
promotes the three independently accepted 1.1 waves as one hardware-validated
Workbench release.

## What changed

### Library shelf, buffers, and compiler architecture

- Boot staging now provides an Attic-backed library shelf, so staged modules
  remain available independently of the work-disk swap.
- Export-only interning and directory-only private entries recover session
  symbol and name-pool capacity.
- First-class mutable byte Buffers provide fixed-length binary storage,
  zero-based access, String conversion, and a bulk transport seam.
- The compiler tier is no longer permanently resident. It is loaded from the
  shelf when needed and retired without leaking compiler-private references.
- `filter` is delivered and covered by surface/dispatch parity; the historical
  live-document erratum is retired.
- The REPL now starts with the lisp65 block-lambda banner.

### Transactional persistent compilation and language surface

- `compile-string` and editor compilation save arbitrary library names through
  M65D's copy-on-write transaction. Preallocated `fasl*` slots are gone.
- Staging writes are verified, directory publication is last, and the mounted
  medium remains identity-bound throughout the transaction.
- `read-from-string` is delivered.
- List primitives share one implementation path with strict-arity and
  direct/`funcall`/`apply` parity.
- The existing L65E-v1 overlay supplies readable text for the 43 error codes
  reachable by the Workbench, with the allocation-free `Ehh` fallback.
- Function metadata proves exact arity for 101 entries. Thirty-four native or
  macro entries remain explicitly unresolved; 1.1 does not claim complete
  integrated help.

### Generated IDE keymap

The editor keymap, dispatcher data, generated documentation, and tests now come
from one registry. The release exposes 41 bindings and fixes:

- sticky `C-x` prefix state;
- multi-digit `goto-line` truncation;
- two-character command-launcher prefix acceptance;
- the unreachable mark binding, now `C-x Space`;
- conflicting editor uses of RUN/STOP.

`C-x C-c` is the sole editor exit and preserves the active buffer. RUN/STOP is
reserved for aborting evaluation; while the REPL is idle it has no product
action. The authoritative table is the
[generated keymap](../generated/ide-keymap.md).

## Upgrade notes from 1.0.1

- Keep the same BASIC-first boot and one-drive swap workflow.
- Continue to load IDE, optional IDEX, and M65D while `L65SYS` is mounted.
- Do not provision `fasl0`--`fasl2`. Compile to an arbitrary new library name
  on the mounted work disk.
- Use the generated 1.1 keymap. The 1.0.x keyboard erratum no longer describes
  the current product.
- Platform Reset returns to BASIC. For a fresh lisp65 session, restart from the
  product disk; power-cycle only when a fully cold Attic state is required.

## Known limitations

- The first call after a persistent definition takes about **1.90–1.96
  seconds** on the reference MEGA65; isolated longer observations exist. Warm
  expressions take about **0.20 seconds**. This remains a documented
  limitation, not a passed performance target. Enter related definitions as
  one block to amortize the reload. C2 direct-Attic execution is the committed
  1.2 cure; the exception does not renew silently if C2 leaves that scope.
- Buffers print as `?`. Use `buffer-ref` and `buffer-length` to inspect them.
- Banner colors persist after scrolling because character RAM scrolls but color
  RAM does not. Text crossing the former banner rows can inherit those colors.
  This is display-only and does not affect program state. `screen-clear` is not
  a workaround because it leaves color attributes unchanged. The fix requires
  the C2 runtime-layout evolution.
- There is no dependency-safe `unload` and no one-command `restart-repl`.
  RUN/STOP aborts evaluation, a product-disk restart creates a fresh session,
  and a power cycle creates a fully cold start. `restart-repl` is committed C2.3
  work.
- M65D/editor saves support at most 8,192 bytes; evaluator `load` has a separate
  38,400-byte staging ceiling.
- The on-device compiler builds Workbench L65M modules, not standalone runtimes
  or bootable application disks.
- The editor still lacks undo/redo, interactive completion, integrated help,
  and full structural editing.
- Xemu remains a prefilter. F011, SD-buffer, Freezer, reset, media-swap, and
  timing claims require hardware.
- One drive is supported and no on-device formatter is supplied.
- The Freezer media-swap residual window can affect at most one already-started
  sector before status 12 stops further writes. Keep backups.
