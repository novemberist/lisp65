# lisp65 1.1 Wave 3 candidate notes

Status: **historical candidate record; superseded by [1.1.0](1.1.0.md)**

Wave 3 makes the IDE keymap, its generated user documentation, and its test
matrix derive from one source. Forty-one bindings are represented, including
the corrected `C-x Space`, exact M-x lookup, multi-digit `goto-line`, a
non-sticky `C-x` prefix, and `C-x C-c` as the only editor exit. RUN/STOP remains
reserved for the global evaluation abort.

The final Wave 3 candidate passed G5 14/14 and all five applicable G6 hardware
cases. Receipt-less pre-smokes and non-authoritative dry variants remained
diagnostic only and did not replace the sealed G5/G6 claims.

## Known issue — banner colors persist after scrolling

The boot banner's colored cells (yellow lambda, gray rule) keep their color
when the screen scrolls; white text scrolling through rows 0–8 appears
color-masked. This is display-only: no data or program state is affected.

The screen driver scrolls character data but not color RAM. The bounded Wave 3
integration attempts proved the color-safe rider mechanics but could not add
the rider under the current runtime-layout grammar. The fix therefore ships
with the C2 runtime-layout evolution.

`screen-clear` is not a workaround: it clears character cells and resets the
cursor but deliberately leaves color attributes unchanged.
