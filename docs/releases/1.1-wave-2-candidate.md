# lisp65 1.1 Wave 2 candidate notes

Status: **historical candidate record; superseded by [1.1.0](1.1.0.md)**

Wave 2 builds on the sealed Wave 1 product with transactional persistent
compilation, `read-from-string`, a unified list-primitive core,
and a pinned function-metadata contract. The final Wave 2 candidate passed its
fresh G5/G6 acceptance and was superseded by the complete 1.1.0 release.

## Persistent compilation

`compile-string` and the editor compiler path no longer require preallocated
`fasl*` files. They save arbitrary library names through the same M65D
copy-on-write transaction as ordinary persistence: verified staging precedes
directory publication, the mounted-medium identity remains bound throughout,
and failures do not publish a partial replacement.

The Wave 2 hardware run must still demonstrate the complete sequence on a real
work D81: compile, remount, load, execute, and independent D81/BAM validation.

## Fresh-session workflow

`restart-repl` is not delivered by Wave 2. The stock MEGA65 platform reset
returns to BASIC, so a fresh 1.1 session requires restarting Lisp65 from the
product disk; a power cycle additionally clears Attic state. Three bounded
pre-C2 designs failed on product semantics, session-mutated Bank 5, and the
resident boundary. The owner assigned the one-command restart and its
double-invocation hardware acceptance to C2.3.

## Error text

The existing L65E-v1 overlay is the canonical error-text carrier. It binds 60
stable codes, provides readable text for 43 Workbench codes, and retains the
resident allocation-free `Ehh` fallback for omitted or unavailable text.

## Metadata claim boundary

The generated index proves exact arity for 102 public entries. Thirty-four
native or macro authorities remain explicitly unresolved. The candidate does
not claim complete integrated help until those entries acquire authoritative
metadata in the C2-era format work.

## Numeric and interactive limitations

Fixnums are signed 15-bit two's-complement values and arithmetic wraps modulo
2^15 without an overflow error. Thus `(fac 8)` yields 7,552 rather than the
mathematical 40,320.

The first expression compiled after a persistent definition normally takes
about 1.90 to 1.96 seconds on the reference MEGA65; occasional longer times
have been observed. Warm expressions take about 0.20 seconds. This remains a
dated exception with C2 direct-Attic execution committed as the 1.2 cure, not a
passed performance target.

Buffers print as the opaque marker `?`; use `buffer-ref` and `buffer-length` to
inspect them.
