# lisp65 1.0.1 Release Notes

Released: 2026-07-15
Language profile: Dialect V2
Release scope: package and documentation correction only

## What changed

Release 1.0.1 corrects the bundled first-session instructions. With one drive,
load all three libraries while `L65SYS` is still mounted:

```lisp
(load-lib "ide")
(load-lib "idex")
(load-lib "m65d")
```

Only then swap to the supplied blank work image or another valid non-product
1581 disk. IDEX provides the command launcher and extended editor operations;
M65D is required for persistence after the swap.

The README, User Guide, and release notes also carry the post-release IDE input
erratum and the current maturity limits.

## What did not change

The 13 product artifacts are byte-identical to release 1.0.0 and retain product
set `c41b9643ada1195f48c384d9d582a3d870a68c4ccc3dee9500dc86a7f009c165`.
No firmware, Workbench PRG, library, D81, keymap, keyboard driver, memory budget,
or hardware behavior changed. Existing G5 and G6 hardware receipts remain valid
because they bind that product set rather than the surrounding package.

The package was rebuilt deterministically, checked for 13/13 product identity,
and statically rebound to the existing G6 receipts. No new power cycle was
claimed or required.

## Persistent compilation remains limited

The blank work image intentionally contains no preallocated `fasl0`--`fasl2`
targets. The legacy fixed-slot compiler writer does not share M65D's complete
product-media denylist or transaction binding. A measured native guard did not
fit the release Bank-0 layout and was rejected rather than forcing a capacity
exception.

Consequently `compile-load` and `compile-buffer-to-lib` report `slot missing`
on the supplied image. Externally provisioned slots remain an expert-only,
non-transactional path; do not change media while compilation is running.
Transactional compiler persistence and removal of fixed slots are planned for
1.1.

## IDE input limitations

Release 1.0.1 retains the 1.0.0 product implementation. Arrow navigation is the
only physical editor input path accepted end to end. `C-Space` is broken,
physical `M-x` is not implemented, and the remaining Control and `C-x` chords
should be treated as experimental. Use `C-x x` or `C-x Return` for the command
launcher when IDEX is loaded, and use `save-buffer-to` from the REPL for the
most reliable save workflow.

## Verification

Run the verifier before using either disk image:

```sh
python3 verify.py
```

The verifier checks the complete 1.0.1 package, the unchanged 13-artifact
product identity, the inherited G6 seal, and the package-only receipt rebinding.
